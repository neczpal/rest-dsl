# Week 9

## Progress:

-   Trait Support:
    -   The DSL now supports traits, which are reusable sets of fields that can be composed into models.
    -   This is achieved using the `trait` keyword to define a trait, and the `+` operator to include it in a model.
    -   The Java generator merges the fields from the trait into the model class.
    -   The OpenAPI generator also merges the trait's properties into the model's schema.

-   New Types Support:
    -   The DSL now supports a wider range of primitive types, including `Long`, `Float`, `Double`, `Boolean`, `Date`, and `DateTime`.
    -   The `TypeMapper` class has been updated to handle the mapping of these new types to their corresponding Java and OpenAPI equivalents.

### Example

**RestDSL:**

```rdsl
traits {
    Timestamped {
        createdAt: DateTime
        updatedAt: DateTime
    }
}

models {
    Product {
        + Timestamped
        id: Long
        name: String
        price: Float
        available: Boolean
        releaseDate: Date
    }
}
```

**Generated Java:**

```java
public interface Timestamped {
  LocalDateTime createdAt();

  LocalDateTime updatedAt();
}

public record Product(
    LocalDateTime createdAt,
    LocalDateTime updatedAt,
    Long id,
    String name,
    Float price,
    Boolean available,
    LocalDate releaseDate
) implements Timestamped {}
```

**Generated OpenAPI Schemas:**

```yaml
    Timestamped:
      type: object
      properties:
        createdAt:
          type: string
          format: date-time
        updatedAt:
          type: string
          format: date-time
    Product:
      allOf:
      - $ref: "#/components/schemas/Timestamped"
      - type: object
        properties:
          id:
            type: integer
            format: int64
          name:
            type: string
          price:
            type: number
            format: float
          available:
            type: boolean
          releaseDate:
            type: string
            format: date
```
