# Week 10

## Progress:

- Maven Plugin Integration:
    - Created a Maven plugin (`rest-dsl-maven-plugin`) to automate the code generation process.
    - The plugin exposes a `generate` goal that reads `.rdsl` files and generates code using the specified generator (e.g., OpenAPI, Java).
    - This allows developers to integrate the REST DSL directly into their build pipelines.
- Example project using maven generation created: https://github.com/neczpal/rest-dsl-examples/
- Minor refactor to support CLI and Maven plugin from the same source
- Minor bugfix java api client generator did not include the service base

### Examples

**pom.xml Configuration for Server interface generation**

```xml
<build>
    <plugins>
            <plugin>
                <groupId>com.example</groupId>
                <artifactId>rest-dsl-maven-plugin</artifactId>
                <version>1.0-SNAPSHOT</version>
                <executions>
                    <execution>
                        <goals>
                            <goal>generate</goal>
                        </goals>
                        <configuration>
                            <inputFile>${project.parent.basedir}/example.rdsl</inputFile>
                            <generatorName>spring</generatorName>
                            <outputDirectory>target/generated-sources/java</outputDirectory>
                            <packageName>com.example</packageName>
                        </configuration>
                    </execution>
                </executions>
            </plugin>
    </plugins>
</build>
```
